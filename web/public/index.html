// privacy by architecture

We can't see your photos.
By design, not by promise.

GitPhotos has no analytics, no SDKs phoning home, and no backend to leak. Your token and your pixels never touch our infrastructure — because we don't run any. The only place your photos go is the GitHub account you signed into.

// questions

Good to know

Do I need a paid GitHub plan? +

No. A free GitHub account includes unlimited private repositories, which is all GitPhotos needs. You're using storage you already have.

How fast is the first backup? +

GitPhotos uses GitHub's standard API, which allows roughly 1,500 photos per hour. Large first-time backups simply continue across sessions until they're done.

What's "auto-sharding"? +

Each repo is capped at about 4.5 GB. When one fills, GitPhotos creates the next private repo automatically. Add more GitHub accounts and it spreads new photos across them — the primary account keeps the index.

Can GitPhotos (or anyone) see my photos? +

No. There's no GitPhotos server. Photos go from your device straight to your own private repos. Your access token is stored only in the iOS Keychain on your phone.

How do I sign in? +

Sign in with a GitHub device code, or paste a personal access token with the repo scope. Tap "How to get a token" in the app for step-by-step instructions.

How do I delete my backups? +

They're just repositories on your account. Delete a repo on GitHub to delete those photos, or revoke the token to cut off access entirely. You're always in control.
